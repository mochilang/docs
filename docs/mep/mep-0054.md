## Abstract

MEP-54 adds a Mochi-to-Go transpiler that lowers a type-checked Mochi program to idiomatic Go 1.22+ source via a structural `gotree` AST, then drives the `go` toolchain to produce statically linked binaries. The pipeline reuses MEP-45's `aotir` IR, monomorphisation, match-to-decision-tree, and closure-conversion passes. The proposal is additive: `mochi build --target=go` routes through this pipeline; all other targets are unaffected.

## Motivation

Go occupies a unique position in the language ecosystem: near-zero deployment overhead (single static binary, no runtime install), first-class goroutine concurrency, `go module` for dependency management, `CGo` for C interop, and a large library of cloud-native tooling (Kubernetes controllers, gRPC, OpenTelemetry). MEP-54 makes Mochi programs available to teams that live in the Go ecosystem.

## Specification

### Lowering map

| Mochi construct | Go output |
|-----------------|-----------|
| `int` | `int64` |
| `float` | `float64` |
| `bool` | `bool` |
| `str` | `string` |
| `list<T>` | `[]T` |
| `map<K,V>` | `map[K]V` |
| `set<T>` | `map[T]struct{}` |
| record | `struct` |
| sum type | tagged `struct` union with `uint8` tag |
| `fun(A) B` | `func(A) B` |
| `let` | `var` (or `:=` short decl) |
| `agent` | `struct` + goroutine + `chan Message` mailbox |
| `chan<T>` | buffered `chan T` with fan-out broadcast helper |
| `async`/`await` | goroutine + channel rendezvous |
| `extern` C | CGo `import "C"` declaration |
| LLM `generate` | cassette replay (SHA-256 of `provider:prompt`) |
| `fetch` | `net/http` |

### Runtime package

`github.com/mochilang/mochi-go` (Apache-2.0) provides: `io` (print/println deterministic formatting), `conv` (int/float/str casts), `strings` (split/trim/contains/index/replace/upper/lower/format), `chan` (broadcast helper), `stream` (fan-out broadcast), `panic` (catch/rethrow), `fetch` (HTTP GET/POST), `json` (encode/decode), `llm` (cassette replay), and `check` (assertion helpers for tests).

### Reproducible builds

`go build -trimpath` plus `SOURCE_DATE_EPOCH=0` in the build environment produces bit-identical binaries for identical Go source on the same toolchain version. The reproducibility gate builds each fixture twice and asserts SHA-256 equality.
